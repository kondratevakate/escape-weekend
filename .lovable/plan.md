
# План: Оптимизация загрузки карты

## Проблема
Карта загружается медленно из-за тяжёлых художественных тайлов Stamen Watercolor и отсутствия индикатора загрузки.

## Решение

### Шаг 1: Заменить тайлы на более быстрые
**Файл:** `src/components/map/MapView.tsx`

Заменить два тяжёлых слоя (watercolor + labels) на один лёгкий:
- **Вариант A:** CartoDB Positron — минималистичный стиль, быстрый
- **Вариант B:** OpenStreetMap — стандартный, надёжный
- **Вариант C:** Оставить watercolor, но убрать второй слой labels

Рекомендую Вариант A (CartoDB Positron) — красивый и быстрый.

### Шаг 2: Добавить индикатор загрузки
**Файлы:**
- `src/components/map/MapView.tsx` — добавить состояние `isLoading`
- `src/components/map/KolaMap.tsx` — показывать спиннер поверх карты

Когда тайлы загружаются — показываем анимированный спиннер. Когда карта готова — убираем.

### Шаг 3: Оптимизировать tile layer
Добавить опции для кеширования и отложенной загрузки:
```
updateWhenIdle: true      // обновлять только когда карта не двигается
keepBuffer: 2             // меньше буфер = меньше запросов
```

## Технические детали

### Изменения в MapView.tsx:
- Заменить URL тайлов на CartoDB
- Убрать второй слой (labels)
- Добавить обработчик события `load` для отслеживания готовности
- Добавить оптимизационные опции

### Изменения в KolaMap.tsx:
- Добавить состояние `isMapReady`
- Показывать компонент загрузки пока карта не готова

## Ожидаемый результат
- Время загрузки карты уменьшится на ~60-70%
- Пользователь увидит индикатор загрузки вместо пустого экрана
- Повторные визиты будут быстрее благодаря кешированию
