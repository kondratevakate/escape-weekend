# Hedonist Odyssey / WoWAtlas — подробное описание проекта

## 1. Что это и какую проблему решает

**Hedonist Odyssey** (публикуется как **WoWAtlas**, домен `wowatlas.lovable.app`) — это коммерческая **white-label платформа кураторских туристических карт**. Текущая инсталляция посвящена **Кольскому полуострову** и работает под слоганом-позиционированием:

> «Кольский без ошибок: куда ехать, когда ехать и как доехать».

### Боль, которую решает
Самостоятельный туризм по нестандартным регионам (Север, Арктика, удалённые территории) сопряжён с **высокой неопределённостью и рисками**:
- неизвестно, какие места реально стоят поездки, а какие — пустая трата дня;
- сезонность критична (полярная ночь, северное сияние, сезон комаров, доступность дорог);
- мало достоверной информации о безопасности, инфраструктуре, питании;
- стандартные сервисы (Google/Яндекс) показывают «всё подряд» без курации и контекста.

### Решение
Платформа объединяет:
1. **Экспертно-курируемую карту мест** с категориями, тегами, сезонностью и предупреждениями.
2. **Слои культурного контекста** — коренные народы (саамы), ЮНЕСКО, рестораны, исторические объекты.
3. **AI-планировщик маршрутов** (`/trip-planner`) — собирает поездку по дням.
4. **Личный «Тайник»** (`/stash`) — сохранённые места с состояниями (someday / planned / visited).
5. **Экосистему креаторов** — местные эксперты публикуют свои карты и услуги.
6. **Telegram-интеграцию** как основной канал доступа, шеринга и монетизации.

### Зачем люди возвращаются
- **Сезонные пере-посещения**: один и тот же регион выглядит по-разному зимой/летом → пользователь возвращается планировать новую поездку.
- **Тайник как личная база**: накопленные сохранения = lock-in. Чем больше сохранил, тем выше переключательский барьер.
- **Социальный шеринг**: пошарил коллекцию/маршрут другу → друг приходит и тоже сохраняет.
- **Реферальная программа**: пользователи генерируют свои промо-ссылки прямо из меню профиля (с toast-уведомлением о копировании).
- **Обновления от креаторов**: добавляются новые авторские слои, продукты, маршруты.
- **AI-планировщик**: каждое новое путешествие = новая сессия планирования.

**North Star метрика**: количество Planned/Shared actions (планирований и шерингов), а не просмотры контента — продукт намеренно оптимизирован под «виральные объекты».

## 2. Архитектура интерфейса

### 2.1 Главная (`/`) — Split-layout в стиле Airbnb
Двухпанельный layout (на мобилке — стек):

**Левая панель (`DiscoverPanel`, 320px)**:
- Тег-фильтры (`TAG_FILTERS`) — быстрая фильтрация по типу контента.
- Список коллекций (кураторские подборки мест).
- Список мест с превью-карточками.
- Кнопка запуска **Explore Mode** (Tinder-подобный свайп).
- Сворачивается кнопкой `PanelLeftClose`.

**Правая панель — карта (`MapView` + `KolaMap`)**:
- Leaflet-карта, центрирована на Кольском (68.0, 34.0), zoom 7.
- Кастомные маркеры по категориям.
- **Переключаемые слои**: Избранное, История/Коренные народы, ЮНЕСКО, Рестораны, Рельеф.
- При клике на маркер — всплывающая `PlaceCard` или `RestaurantCard` снизу.
- Лоадер с спиннером пока карта не готова.
- Empty-state с кнопкой сброса фильтра.

### 2.2 Хедер (`Header`)
- Брендинг Hedonist Odyssey.
- Поисковая строка.
- `LanguageSwitcher` (RU/EN, через `LanguageContext` + `i18n`).
- Меню профиля **адаптивное под роль**:
  - Гость → CTA «получить доступ через Telegram».
  - User → статус доступа, ссылка на Тайник, генератор реферальной ссылки.
  - Creator → доступ к `/creator`.
- Счётчик Тайника (бейдж).

### 2.3 Explore Mode (свайп)
Полноэкранный режим в духе Tinder: свайп вправо → в Тайник, влево → пропустить. Отдельная механика обнаружения новых мест без когнитивной перегрузки списка.

### 2.4 AI Trip Planner (`/trip-planner`)
- Сайдбар с настройками (даты, интересы).
- Колонки по дням (`DayColumn`).
- Результат от AI (`AITripResult`).
- Хук `useTripBuilder`.

### 2.5 Stash / Тайник (`/stash`)
Личное хранилище мест с состояниями:
- **Someday** — захотелось, отложил.
- **Planned** — добавлено в план.
- **Visited** — посещено.
Плюс пользовательские списки (`useUserLists`) — кастомные коллекции с возможностью шеринга.

### 2.6 Платформа креаторов (`/creator` + `/creator/:id`)

**Дашборд `/creator`**:
- Онбординг-чеклист с прогресс-баром (4 шага: био, карта, продукт/ссылка, шеринг).
- Табы: Профиль / Карты / Продукты / Ссылки / Планирование маршрута.
- Управление внешними картами (Google/Yandex/Maps.me/др.).
- Инфопродукты (название, описание, цена, валюта RUB/USD/EUR, ссылка).
- Произвольные ссылки (a-la Taplink).
- Настройка платного планирования маршрута + Telegram username для связи.
- Кнопки шеринга профиля и реферальной ссылки.
- Превью публичной страницы.

**Публичная страница `/creator/:id` — Taplink-style**:
- Градиентный хедер с аватаром (DiceBear) и био.
- Полноширинные карточки-кнопки для карт, продуктов (с бейджами цены), ссылок.
- Высоковидимый CTA «Заказать маршрут» → открывает `https://t.me/{username}`.
- Кнопка «Поделиться» (`navigator.share` с fallback на clipboard).
- Glassmorphism-эффекты, hover-transitions, mobile-first.

### 2.7 Telegram-мост (`TelegramBridgeSheet`)
Bottom-sheet, который появляется после N сохранений в Тайник (хук `useTelegramCTA`). Конвертирует анонимного пользователя в пользователя Telegram-бота — основной канал ретеншна и нотификаций.

### 2.8 Welcome / Cookie / Auth
- `WelcomeModal` — первое знакомство с 4 фичами (Explore Map, Swipe Mode, Collections, Stash).
- `CookieConsent` — GDPR-баннер, отдельная страница `/cookie-policy`.
- `LoginModal` (через `AuthContext`) + `LockedScreen` / `PremiumGate` для платного контента.

## 3. Визуальный язык

- **Крафтовая эстетика в тёплых тонах**: основной акцент — оранжевый, фон — светлый бежевый (намеренно нейтрализованный, чтобы не конкурировать с картой).
- **Стиль Airbnb**: split-layout, чистые карточки, скруглённые углы.
- **Без backdrop-blur** на оверлеях — сплошная заливка, чтобы карта-подложка читалась чисто.
- Shadcn/ui как базовая библиотека компонентов, Tailwind для стилей.
- Mobile-first, проверено на 375×812.

---

## 4. Ролевая модель

Многоролевая клиентская модель (определяется через URL `?role=creator` или метаданные Telegram):
- **user** — промокоды, рефералки, Тайник, AI-планировщик.
- **creator** — личный кабинет, публикация карты в общем слое «Авторы», публичная Taplink-страница.
- **admin** — системное управление.

Роли используются для **UI-гейтинга**, не для безопасности (это клиентская демо-модель; для прод-уровня требуется server-side validation через Supabase user_roles + RLS).

## 5. Стек

- **React 18 + Vite 5 + TypeScript 5 + Tailwind v3**.
- **shadcn/ui** (Radix-based) + **lucide-react**.
- **Leaflet** для карты, кастомные слои данных в `src/data/*Layer.ts`.
- **React Router** (роуты выше).
- **TanStack Query** (готов, активно ещё не используется — данные локальные).
- **localStorage** для персистентности (Тайник, списки, профиль креатора, онбординг).
- **i18n RU/EN** через собственный `LanguageContext`.
- Backend-ready под **Lovable Cloud / Supabase**, сейчас работает как клиентский MVP.

---

## 6. Бизнес-модель и виральные петли

**B2C**: фримиум-доступ через Telegram-бот, премиум-гейтинг отдельных коллекций (`PremiumGate`), реферальная программа.

**B2B / White-label**: регионы и турбизнес покупают инсталляцию платформы под свою территорию.

**Creator economy**: креаторы монетизируют инфопродукты и платное планирование маршрутов прямо через свою публичную страницу → Telegram-чат.

**Виральные петли**:
1. Шеринг места → friend opens → saves → invited to Telegram.
2. Шеринг коллекции (`CollectionShareButton`) → подписка на автора.
3. Шеринг плана поездки → совместное планирование.
4. Реферальная ссылка из меню → промокод-бонус.
5. `useSharePostcard` — генерация красивых открыток-превью для соцсетей.

## 7. Что важно сказать ревьюеру

- Это **MVP в стадии product-market fit поиска** на конкретном регионе (Кольский), архитектурно готовый к white-label-тиражированию.
- Сильные стороны: чёткое позиционирование, виральные механики «зашиты» в продукт, экосистема креаторов с реальной монетизацией, продуманный mobile UX.
- Точки роста: данные пока статичные (TS-файлы), нет настоящего бэкенда (готовность к Supabase есть), роли клиентские, нет аналитики ретеншна, AI-планировщик в раннем виде.
- Ключевой канал: **Telegram** — и как auth, и как нотификации, и как монетизация креаторов.
