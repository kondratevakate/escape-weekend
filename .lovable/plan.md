
# План: Социальные метрики + Шеринг (минималистично)

## Концепция

Вдохновляясь Wanderlog, добавляем легкую социальную валидацию и шеринг, но без перегруза интерфейса. Акцент на "курированной подборке лучших мест" - минималистично и со вкусом.

---

## Часть 1: Социальные метрики (мок-данные)

### 1.1 Расширяем данные мест
**Файл:** `src/data/kolaPlaces.ts`

Добавляем поля для социальных метрик (пока мок):
- `likesCount` - сколько людей добавили в избранное
- `sharesCount` - сколько раз поделились

Эти данные генерируются детерминистично на основе ID места (для консистентности без бэкенда).

### 1.2 Хук для работы со статистикой
**Файл:** `src/hooks/usePlaceStats.ts`

Создаем хук, который:
- Возвращает мок-статистику для каждого места
- Готов к переключению на реальный API позже
- Трекает локальные шеры (в localStorage)

## Технические детали

### Web Share API
На мобильных устройствах используем нативный шеринг:
```typescript
if (navigator.share) {
  await navigator.share({
    title: place.name,
    text: `📍 ${place.name} - Кольский полуостров`,
    url: googleMapsUrl,
  });
}
```

### Мок-данные социальных метрик
Генерируем детерминистично на основе ID:
```typescript
const getLikesCount = (placeId: string) => {
  const hash = placeId.split('').reduce((a, b) => a + b.charCodeAt(0), 0);
  return 50 + (hash % 200); // 50-250 лайков
};
```

### localStorage для отслеживания shares
Сохраняем ID мест которыми поделились, чтобы показывать "+1" после шеринга.

## Сохраняем минимализм

- Метрики показываем компактно (одна строка)
- Цифры округляем (2.3K вместо 2347)

- Фокус на "курированности" - это отобранные лучшие места
