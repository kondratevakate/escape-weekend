

# План: Добавить функционал "Сохранить в список"

## Текущее состояние

- Есть система лайков (избранное) — простой массив `favorites[]` в localStorage
- Есть готовые коллекции (`collections.ts`) — но они статичные, заданы вручную
- Есть UI-компоненты: `Drawer` (vaul) для мобильного bottom-sheet

## Что реализуем

Паттерн в стиле **Pinterest / Google Maps / Airbnb**:

При клике на иконку закладки (Bookmark) открывается bottom-sheet с:
1. Списком пользовательских коллекций
2. Возможностью создать новую коллекцию
3. Выбором в какие списки добавить место

## Что изменится

### 1. Новый хук `useUserLists`

Управление пользовательскими списками в localStorage:

```tsx
interface UserList {
  id: string;
  name: string;
  emoji: string;
  placeIds: string[];
  createdAt: number;
}

// Хук возвращает:
{
  lists: UserList[];
  createList: (name: string, emoji: string) => string;
  deleteList: (id: string) => void;
  addToList: (listId: string, placeId: string) => void;
  removeFromList: (listId: string, placeId: string) => void;
  isInList: (listId: string, placeId: string) => boolean;
  getListsForPlace: (placeId: string) => UserList[];
}
```

### 2. Новый компонент `SaveToListDrawer`

Bottom-sheet drawer (на базе vaul) с:
- Список пользовательских коллекций с checkbox
- Кнопка "Создать список" → показывает поле ввода
- Выбор эмодзи для нового списка

### 3. Изменения в PlaceCard

- Добавить кнопку 🔖 (Bookmark) рядом с лайком
- При клике открывать `SaveToListDrawer`

### 4. Отображение списков в DiscoverPanel

- Добавить секцию "Мои списки" в боковую панель
- Показывать пользовательские коллекции рядом с системными

## Техническая реализация

### Структура данных в localStorage

```tsx
// key: 'kola_user_lists'
[
  {
    id: 'list_1706745600000',
    name: 'Хочу посетить',
    emoji: '🏔️',
    placeIds: ['teriberka', 'khibiny'],
    createdAt: 1706745600000
  }
]
```

### Дефолтные списки

При первом запуске создаются:
- 🏔️ Хочу посетить (Want to visit)
- ⭐ Избранное (Favorites) — синхронизируется с текущими favorites
