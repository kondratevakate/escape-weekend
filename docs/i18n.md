# i18n

WoWAtlas is bilingual: **RU** (default) and **EN**, equal first-class citizens.

## Adding a new translation key

1. Open `src/lib/i18n.ts`.
2. Add the key to **both** `ru` and `en` blocks. Same key string, different value.
3. Use it: `const { t } = useLanguage(); return t('my.new.key');`

For short one-off conditionals (rare), inline `language === 'ru' ? '…' : '…'` is acceptable.

## Adding a new language

1. Add the new locale (e.g. `de`) to the `Language` type in `src/contexts/LanguageContext.tsx`.
2. Duplicate the `ru` block in `i18n.ts` and translate **every key by hand**.
3. Add the language option to `src/components/LanguageSwitcher.tsx`. Use language codes (DE), **not flag emojis**.
4. Add an entry to `BRAND.md` and `CONTRIBUTING.md` so future contributors know.

## Hard rules

- ❌ **No machine translation.** Brand voice depends on humans. If you can't write it natively, find someone who can.
- ❌ **No flag emojis.** Use language codes (RU / EN / DE). Flags politicize neutral content.
- ❌ Don't ship a partial translation. If a language is missing 10% of strings, finish them or don't ship.

## Voice across languages

Read [`BRAND.md`](../BRAND.md) before translating. The 4 voice principles (warm, concrete, honest, bilingual) apply to **every** language. Don't let translation drift toward generic travel-brochure tone.
